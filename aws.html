Amazon Web Services Practicals

One of my tasks involves gaining practical experience in creating EC2 instances, starting from selecting the appropriate AMI to deploying the instances as virtual machines in the cloud. This project aims to familiarize me with the daily tasks of a DevOps Engineer, focusing on cloud architecture, data migration, and system patching or deployment.

As part of this project, I have also been assigned the responsibility of configuring inbound and outbound rules for the instances I've created. This crucial step is implemented to safeguard all company data and hosted applications from potential threats, including unwanted attacks and malware. To further strengthen security measures, I have been creating and utilizing security groups, ensuring that only authorized traffic is allowed for EC2 instances within the company's network.

To maintain a safe and cost-effective environment, I have conducted this project within the AWS Playground, minimizing any potential risks to real company data while still gaining valuable hands-on experience in a controlled setting.
